# Azure Observability & Monitoring Project

## ?? Overview
This project demonstrates an end-to-end monitoring and observability system using Azure Monitor, Log Analytics, and Grafana by simulating real-time application logs.

## ?? Architecture
Python Script ? Azure Log Analytics ? KQL Queries ? Azure Dashboards ? Alerts ? Grafana Dashboard

## ?? Features
- Real-time log generation using Python
- Log ingestion into Azure Log Analytics
- Monitoring using KQL (error rate, latency, traffic)
- Azure dashboard creation
- Alerting for error spikes and latency issues
- Grafana dashboard with advanced visualization
- Incident simulation and root cause analysis

## ??? Tools Used
- Azure Monitor
- Log Analytics Workspace
- Grafana
- Python
- KQL (Kusto Query Language)

## ?? Key Metrics
- Error Rate (%)
- Response Time (ms)
- Request Count

## ?? Alerts Implemented
- High Error Rate Alert (>5%)
- High Latency Alert (>500ms)

## ?? Grafana Dashboard Panels
- Error vs Success (Pie Chart)
- Slow Requests Trend
- Latency Distribution
- System Health (Error Rate %)

## ?? Incident Simulation
Simulated system failures to validate alerts and monitoring effectiveness.

## ?? Outcome
Built a real-time monitoring system similar to production environments, improving observability and proactive issue detection.
